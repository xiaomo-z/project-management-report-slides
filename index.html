01 / 项目背景

需求来源：毕业纪念、班级凝聚与安全可控

为什么要做这个项目？

毕业阶段同学存在共同纪念需求，但自由出行容易出现预算不透明、行程不统一、风险无人负责等问题。

因此，需要将班级旅行转化为一个有目标、有范围、有计划、有验收的项目。

纪念形成共同毕业记忆
协同提升班级凝聚力
控制安全、预算、时间可控
理论依据：项目启动 / 需求识别 / 利益相关者

01B / 干系人分析

先识别“谁会影响项目”，再设计沟通与约束

班级同学核心参与者关注：预算、体验、安全、自由度
班委/项目组组织执行者关注：分工清晰、进度可控、责任可追踪
家长/辅导员安全监督者关注：出行安全、应急预案、联系人机制
供应商资源提供者关注：人数确认、付款节点、服务边界
管理重点：把不同干系人的诉求转化为项目约束：预算上限、安全底线、时间节点、信息发布频率。
理论依据：项目利益相关者 / 需求收集 / 沟通计划
这一页用于说明项目不是只服务“旅行好玩”，还要平衡同学、班委、家长/辅导员、供应商等多方诉求。

02 / 项目目标与范围

目标清晰，边界明确，避免范围蔓延

项目目标组织一次杭州出发、预算可控、安全有序的安吉两日一夜毕业旅行
做什么路线规划、交通住宿、活动安排、预算控制、通知执行、复盘总结
不做什么不安排高风险户外项目；不做长途跨省旅行；不超出集体预算上限
交付物行程方案、预算表、风险清单、执行手册、照片视频素材、复盘报告
理论依据：项目范围管理 / 产品范围 / 项目范围 / 范围控制
重点讲“做什么”和“不做什么”，体现范围边界，避免范围蔓延。

03 / 方案选择

目的地方案：湖州安吉两日一夜

杭州安吉

以“安全可控、预算适中、体验完整、组织难度低”为原则，选择湖州安吉作为毕业旅行目的地。

交通可控杭州出发距离近，适合包车往返，减少跨城换乘和迟到风险。
体验均衡自然景观、团建活动、毕业合影场景兼具，满足多数同学需求。

理论依据：项目可行性分析 / 多方案比较 / 资源约束 / 方案优选
这一页重点讲“为什么选安吉”。不要只说安吉好玩，而是从交通、体验、预算和风险四个项目约束角度证明它更适合作为项目方案。

06B / 执行与控制

不是“做完计划就结束”，而是持续跟踪偏差

进度跟踪每个里程碑设置负责人和完成证明，延期超过 1 天进入协调处理。
成本跟踪按交通、住宿、餐饮、活动四类记录实际支出，与预算基线比较。
质量跟踪用签到率、准点率、满意度、资料完整性判断项目结果。
风险跟踪天气、交通、人员变化每日更新，触发条件出现即启用备用方案。
理论依据：项目执行过程 / 项目控制过程 / 偏差纠正
强调项目管理的关键在控制过程：发现偏差、分析原因、采取纠正措施。

07B / 偏差监控

用“简化挣值”说明进度与成本是否健康

PV计划价值按计划本阶段应完成的工作量/预算
EV挣值实际已完成工作的计划价值
AC实际成本已经发生的真实支出
SV = EV - PV判断进度是否滞后
CV = EV - AC判断成本是否超支
SPI / CPI用于综合判断执行效率
理论依据：挣值管理 EVM / 进度偏差 SV / 成本偏差 CV
这里不需要真的计算复杂数据，主要展示我们知道如何用教材中的方法进行过程监控。

08 / 质量管理

验收标准：旅行体验也要可衡量

安全性全员购买保险；出行签到率 100%；无重大安全事故
准时性关键节点延误不超过 30 分钟；备用方案可启动
满意度活动后问卷满意度目标 ≥ 85%
预算控制总支出不超过预算基线 10%
资料完整通知、名单、票据、照片视频、复盘报告归档
理论依据：项目质量管理 / 质量计划 / 质量控制 / 验收标准

10 / 沟通协作

沟通计划：让信息在正确时间到达正确的人

项目组会议每周一次，确认计划偏差与待办
班级通知群发布报名、缴费、集合、注意事项
现场执行群实时定位、签到、突发事件处理
复盘文档费用公示、素材归档、经验总结
冲突处理原则：预算、时间、路线分歧优先采用“正视 + 妥协”，以项目目标和安全边界为判断依据。
理论依据：沟通计划 / 信息发布 / 冲突管理

11 / 收尾复盘

项目收尾：从完成活动到沉淀经验

1成果验收行程完成、预算公开、成员反馈达标
2资料移交票据、名单、通知、照片视频统一归档
3经验复盘总结计划偏差、风险处理与协作问题
4后续优化形成班级活动项目模板，服务后续活动
理论依据：项目收尾 / 项目验收 / 后评价
